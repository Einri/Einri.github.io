使用Hexo+GitHub搭建博客的过程

发表于 2018-08-11   |   更新于 2018-09-20 | 分类于 学习
字数: 1,400字 | 时长 ≈ 5分钟

前言

       第一次使用Hexo+GitHub搭建博客，之前从未接触过，比之WordPress要麻烦很多，需要操作的软件也比较多，看了很多的教程，有的详细有的简单，但多多少少都缺点儿什么，尤其是像我这种菜鸟级别的，很多时候看的一脸懵逼，东拼西凑的看了很多版本的教程，才算是把这个博客搭建起来了，也仅仅是搭建起来，后面还有很多需要学习的地方。

软件安装

       需要用到的包括Git、Node.js、GitHub账号等。

  1. 去Node.js官网下载最新版本的Node.js。注意：需要下载与自己电脑系统相应的版本，过程默认安装就可以，无需更改任何选项。
    安装结束后，通过Windows+R快捷键点开运行，输入cmd，点击确认。

    在命令行内输入【node -v】【npm -v】，如图显示相应的版本号，即安装成功。

  2. 安装Git和配置好Git的环境。下载 桌面空白处 → 右击鼠标 → 如下图所示（安装成功）：

    运行 cmd 可得：

  3. 注册GitHub，新建项目（New repository ），项目必须要遵守格式：账户名.github.io ，不得另起名字（当时因为一时脑抽，选择了一个别的名字，导致后期链接ssh的时候出现了很多莫名其妙的错误）。勾选Initialize this repository with a README 。（忽略那个红色的提醒，是因为建过同名的项目）

  4. 安装Hexo，找个合适的盘建立新的文件夹，用以存放Hexo的安装文件。通过命令行进入该文件夹：

    运行【npm install hexo -g 】命令，安装Hexo。

    运行【hexo -v】，检查安装是否成功，如图所示即成功：

  5. 初始化该文件夹，运行中输入【hexo init 】

  6. 安装需要的组件，桌面空白处，右键点击【Git Bash Here】，进入文件夹：

    输入【npm install】，安装所需要组件，执行完毕后，输入【hexo g】，生成静态页。

  7. 现在我们的本地hexo已经安装完毕，可以启动本地服务器，正式体验了。
    输入【hexo s】开启服务器。如下图所示，就已经成功了，访问：http://localhost:4000/ 可以了。
    PS：每次需要本地调试查看效果的时候，都需要执行这一步。

  8. 看到如下样式，即安装成功。

连接GitHub

       本地调试已经成功，那么现在我们需要将本地的Hexo与GitHub连接起来， 实现由本地推送至GitHub。

  1. 设置git的 【user name】和【email】，鼠标右键点击【Git Base Here】，进入存放Hexo的文件夹，执行【git config –global user.name “einri”】和【git config –global user.email “871150479@qq.com“】。上面两行命令中，”einri”和”871150479@qq.com“需要换成你自己的用户名和邮箱。

  2. 检查git安装后，是否存在.ssh文件夹，里面是否有文件。输入【cd ~/.ssh】进入文件夹，可以进入就说明存在该文件夹，然后输入【ls】查看里面的文件。有【id_rsa】【id_rsa.pub】【known_hosts】这三个文件夹。

    PS:如果没有.ssh这个文件夹，则需要你去生成。没有文件显示如下图：

    输入【ssh-keygen -t rsa -C “你的邮箱” 】，按回车，一直按…直到显示如下图所示即生成成功：

    这时，.ssh文件（默认路径：C:\Users\Administrator）就已经生成成功，输入【cd ~/.ssh】，进入文件夹，看看是不是已经成功进入了呢？

    但是，出现了一个新的问题，本应该是3个文件，可这里却显示了两个，少一个【known_hosts】，这时如果直接操作GitHub连接ssh还是会出错，所以我们需要生成这个文件，输入【ssh -T git@github.com 】，这个命令本来是用于测试ssh是否连接成功的，但在这里我们用来生成【known_hosts】。回车之后，输入【yes】，此时，3个文件全部生成。

  3. 输入【eval “$(ssh-agent -s)”】，添加密钥到ssh-agent

  4. 输入【ssh-add ~/.ssh/id_rsa】，添加生成的SSH key到ssh-agent

  5. 找到C:\Users\Administrator\ .ssh 里面的【id_rsa.pub】文件，用记事本打开，复制全部内容。我们之前注册和新建项目的GitHub，此时就派上用场了。登录GitHub，选择头像下面的【settings】→【SSH and GPG keys】→【New SSH key 】，将【id_rsa.pub】文件里的内容复制上去。


  6. 输入【ssh -T git@github.com】，这是第二次输入这段命令，之前是因为需要生成文件，此时，才是真的用来测试ssh是否连接成功，当在下图中看到自己的用户名，添加ssh成功。

  7. 配置Deployment，在本地Hexo文件夹根目录中，找到_config.yml文件，修改deploy的信息。在该文件的最后一行。

博客部署

       本地Hexo安装成功，以及GitHub的SSH连接也已成功。现在开始新建一篇文章，同步至GitHub。

  1. 新建文章，执行命令【hexo new post “测试”】

  2. 在部署之前需要安装一个扩展：【npm install hexo-deployer-git –save】

  3. 之前新建过一篇名为《测试》的文章，以此为例，先在本地测试查看，执行【hexo s】，通过 http://localhost:4000/ 这个地址访问，如需下一步操作，需要【Ctrl + C】这个组合键，来关闭服务器。

  4. 输入【hexo clean】清除缓存：

  5. 输入【hexo g】生成文件，之后执行【hexo d】上传至GitHub。然后通过 【https://用户名.github.io】来访问博客即可。
